## How to use the search_pdf tool effectively

The `search_pdf` tool performs semantic search over the loaded PDF document. It takes a `query` string and returns the most relevant text passages.

### Crafting good queries
- Write queries that target the specific information you need. Use concrete terms, entities, and financial concepts from the question.
- Include relevant context in your query: company names, fiscal years, metric names, line-item labels, and units.
- Prefer the terminology likely used in the source document (e.g., "total revenue" instead of "how much money did they make").

### When a search does not return relevant results
If the results from `search_pdf` are not helpful, DO NOT repeat the same query. Instead, reformulate your query using these strategies:
1. **Use synonyms or alternate terminology.** Financial documents may use different labels (e.g., "net sales" vs. "revenue", "cost of goods sold" vs. "cost of revenue", "operating income" vs. "income from operations").
2. **Broaden or narrow scope.** If a specific query returned nothing, try a broader concept. If a broad query returned too much noise, add specifics like the year, segment, or line item.
3. **Incorporate context from earlier observations.** Use names, section headings, or table labels you already found to refine the next search. For example, if you found a section titled "Segment Information", search within that context.
4. **Search for the table or section, not the answer.** Instead of querying for a computed value, search for the table or disclosure that would contain the raw data you need (e.g., "consolidated balance sheet 2023" or "notes to financial statements lease obligations").
5. **Break complex questions into parts.** If the question requires multiple data points, search for each one separately rather than putting everything into a single query.
